# Workshop Materials

## Title
From CV to Academic Website:
An AI-Assisted Workflow for Graduate Students

## Audience
Graduate students, doctoral researchers, and early-career scholars who want to turn a traditional CV into a polished academic website and maintain it over time.

## Format
- Duration: 90-120 minutes
- Mode: Hands-on workshop
- Prerequisite knowledge: basic web literacy, basic familiarity with GitHub is helpful but not required

## Workshop Goal
Students will learn how to move through a repeatable cycle:

1. Start from an existing academic CV.
2. Translate CV content into a structured researcher website.
3. Improve the site iteratively with AI support.
4. Enrich publication records with journal covers, abstracts, and detail pages.
5. Publish the site through GitHub Pages.
6. Maintain the site with future CV updates.

## Learning Outcomes
By the end of the workshop, participants will be able to:

- Explain the difference between a static CV and a narrative academic profile site.
- Convert scholarly information into web-friendly sections such as overview, publications, grants, teaching, and service.
- Use AI prompts strategically to revise, expand, and refine academic web content.
- Add publication-level enhancements such as journal cover images and abstract previews.
- Deploy a static academic site on GitHub Pages.
- Plan an update workflow so the site can be revised whenever the CV changes.

## Core Case
This workshop uses a real workflow cycle:

- Start with a scholar CV in Word format.
- Build a GitHub Pages academic profile site.
- Curate publication cards and service sections.
- Add real journal cover images.
- Create abstract previews plus independent publication detail pages.
- Push changes to GitHub and publish the site.

## Suggested Agenda

### Part 1. Why an Academic Website Matters (10 minutes)
- Compare a traditional CV with a public-facing academic website.
- Discuss how a website supports visibility, networking, student recruitment, and grant storytelling.
- Frame the website as a living research profile rather than a static document dump.

### Part 2. The Workflow Cycle (15 minutes)
- Input: CV, profile image, publication metadata, journal covers.
- Transformation: AI-assisted structuring, visual curation, section design, publication formatting.
- Output: publishable academic website.
- Iteration: update the site when new papers, grants, or roles appear.

### Part 3. Guided Walkthrough of the Site Structure (15 minutes)
- `index.html`: overall page structure and section layout
- `styles.css`: visual identity, layout, and typography
- `app.js`: structured content and rendering logic
- `publication.html` and `publication.js`: independent publication detail pages
- `assets/`: CV file, headshot, and journal cover images

### Part 4. Hands-On Activity 1: Converting CV Content into Website Sections (15 minutes)
Students identify which CV content belongs in:

- Overview
- Publications
- Grants
- Teaching
- Service
- Contact and affiliations

Prompt:
"Take this CV content and reorganize it into a concise academic website structure for graduate-level readers. Keep the tone professional but approachable."

Discussion prompt:
"What should stay in the downloadable CV, and what should be surfaced directly on the homepage?"

### Part 5. Hands-On Activity 2: Designing Publication Cards (15 minutes)
Students examine how a publication list becomes a richer research showcase.

Key design choices:
- journal cover image
- title
- authors
- venue
- abstract preview
- link to detail page

Prompt:
"Turn this publication citation into a website card with a journal cover, short abstract preview, and a more detailed article page."

Mini reflection:
"How does visual presentation change the way visitors understand research quality, theme, and identity?"

### Part 6. Hands-On Activity 3: AI-Assisted Iteration (15 minutes)
Students practice prompting for specific revisions.

Example prompt categories:
- layout refinement
- publication updates
- author-role corrections
- service section redesign
- visual benchmarking against academic websites
- deployment and GitHub Pages publishing

Sample prompts:
- "Add a corresponding author label to this paper."
- "Replace placeholder publication thumbnails with journal front covers."
- "Show only a 4-5 line abstract preview in the card and move the full abstract to a detail page."
- "Make the site feel more like a formal academic homepage."

Teaching point:
AI is most useful when the request is concrete, incremental, and tied to a visible output.

### Part 7. Hands-On Activity 4: Publishing and Maintenance (10 minutes)
- Push the project to GitHub.
- Enable GitHub Pages from the main branch.
- Confirm the site URL.
- Discuss update routines for future CV changes.

Prompt:
"Keep the current site structure, but update it when a new CV is uploaded so new publications and profile changes are reflected without rebuilding from scratch."

## Facilitator Notes

### What to Emphasize
- This is not just web design; it is academic identity design.
- The website should not duplicate the CV exactly.
- AI works best as a collaborator for refinement, not as an unquestioned author.
- Publication pages are especially valuable because they support discoverability and narrative framing.

### Common Student Misconceptions
- "I should just upload my CV PDF."
  Response: A CV is comprehensive; a website is curated and navigable.

- "AI should generate everything in one shot."
  Response: Better results usually come from iterative prompting and review.

- "A polished website requires advanced coding."
  Response: A static site with clear structure can be powerful and manageable.

## Workshop Discussion Questions
- What should an academic website foreground for different audiences such as students, collaborators, or grant reviewers?
- How can publication visuals shape first impressions of a research agenda?
- What are the risks of relying on AI for academic self-presentation?
- How can researchers keep their sites current without turning maintenance into a burden?

## Suggested Deliverables for Students
By the end of the session, each student should produce:

- a draft homepage structure
- a short research overview paragraph
- a publication section plan
- one example publication card
- one publication detail page concept
- a simple GitHub Pages publishing plan

## Optional Follow-Up Assignment
Ask students to complete the full cycle after the workshop:

1. Select their current CV.
2. Build or revise a static academic website.
3. Add at least five publication cards.
4. Add one detailed publication page with abstract.
5. Publish the site online.
6. Write a short reflection on what AI helped with and what still required human judgment.

## Short Presenter Script
You can introduce the session with:

"Today we are going to treat an academic website not as a technical side project, but as a scholarly communication tool. We will begin with a conventional CV, transform it into a public-facing research profile, improve it iteratively with AI support, and publish it as a maintainable website. The key idea is not simply to make a site, but to build a repeatable workflow for updating and presenting academic work with intention."

## Quick Prompt Bank

### Structure
"Reorganize this CV into a clear academic website structure with overview, publications, grants, teaching, and service."

### Tone
"Rewrite this biography for an academic homepage aimed at graduate students, collaborators, and external visitors."

### Publication design
"Turn these citations into visually rich publication cards with journal branding and brief summaries."

### Abstract handling
"Show only a short abstract preview on the listing page and move the full abstract to a detail page."

### Maintenance
"Preserve the existing website structure, but update the publication data when a new CV file is added."

### Deployment
"Prepare this static site for GitHub Pages deployment and explain the publish steps."
